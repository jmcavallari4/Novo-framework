# SOP — Customer Journey Map
**Founder OS | Etapa 4 — Definición de la Solución | Agente A09**

---

## Qué es y para qué sirve

El Customer Journey Map visualiza en una línea de tiempo la experiencia completa del cliente desde su perspectiva. No es técnico — es empático. Permite ordenar lo aprendido en entrevistas, entender cómo el cliente vive el problema, detectar momentos críticos de fricción, e identificar oportunidades reales de mejora.

El análisis se construye en tres partes que se complementan:

1. **CJM Actual** — el recorrido del cliente con las soluciones que usa hoy.
2. **CJM Futuro** — el recorrido del cliente con la nueva solución.
3. **Comparativa** — evidencia del delta entre antes y después.

Si la comparativa no es convincente, la solución no está suficientemente diferenciada.

## Antes de mapear el CJM Futuro: definir el tipo de solución

El tipo de solución determina las etapas típicas del CJM Futuro. Preguntarlo antes de arrancar el Paso 2.

| Tipo de solución | Etapas típicas |
|---|---|
| **SaaS** | Descubrimiento → Registro → Onboarding → Primer uso → Uso frecuente → Soporte → Upgrade/Retención |
| **AaaS (Agents as a Service)** | Identificación → Prueba → Resultados → Adaptación → Confianza → Reutilización |
| **Marketplace** | Descubrimiento → Búsqueda → Comparación → Compra → Evaluación |
| **App Mobile** | Descubrimiento → Instalación → Registro → Primer uso → Retención |
| **Consultoría/Servicios** | Contacto → Diagnóstico → Propuesta → Ejecución → Evaluación |
| **Formación sincrónica** | Descubrimiento → Inscripción → Participación → Implementación → Comunidad |
| **Formación asincrónica** | Descubrimiento → Acceso → Consumo → Implementación → Feedback |
| **Comunidad/Plantillas** | Descubrimiento → Acceso → Uso → Implementación → Feedback |

Si el tipo no está claro, el Lean Canvas es el mejor punto de partida.

## Los 3 pasos del CJM

---

### Paso 1 — CJM Actual

Diagnosticar el recorrido que el cliente hace HOY para resolver el JTBD priorizado, usando las soluciones disponibles actualmente.

**Definir el scope del mapa:**
- **Inicio:** el evento desencadenante (qué activa la necesidad). Viene del consolidado.
- **Fin:** el resultado deseado (qué considera "haberlo resuelto"). Viene del JTBD priorizado.

**Identificar 4–7 etapas clave** — derivarlas de lo que los entrevistados describieron, no inventarlas.

**Para cada etapa, mapear 7 dimensiones:**

| Dimensión | Qué captura |
|---|---|
| **Tareas** | Todas las acciones que el cliente realiza, por más pequeñas que sean |
| **Emociones** | Cómo se siente (excitado, confiado, frustrado, ansioso, decepcionado) |
| **Expectativas** | Qué espera de esa etapa (rapidez, seguridad, facilidad, bajo costo) |
| **Solución/herramienta actual** | Qué usa hoy para ejecutar esa etapa |
| **Problemas y fricciones** | Qué consume tiempo, es difícil, propenso a errores, costoso o repetitivo |
| **Brecha expectativa/realidad** | ¿Lo que obtiene está alineado a lo que esperaba? |
| **Oportunidades de mejora** | Qué podría hacerse más rápido, fácil, barato o mejor |

**Al cerrar el Paso 1, identificar:**
- **La etapa más crítica:** donde NO hay que fallar y donde MÁS valor hay que generar. Generalmente es la etapa con más fricciones y mayor impacto emocional negativo.
- **Las fricciones más dolorosas:** señalarlas con citas textuales de entrevistas cuando existan.

**Regla de oro:** cada fricción, emoción y oportunidad debe poder respaldarse con evidencia del consolidado. Si no hay evidencia, señalar que es una hipótesis pendiente de validar.

---

### Paso 2 — CJM Futuro

Diseñar el recorrido del cliente con la nueva solución. El punto de partida son las etapas típicas del tipo de solución elegido, adaptadas al caso específico.

**Para cada etapa, mapear 6 dimensiones:**

| Dimensión | Qué captura |
|---|---|
| **Tareas** | Cómo se ejecuta cada tarea con la nueva solución |
| **Expectativas** | Qué espera el cliente de esa etapa con la nueva solución |
| **Ejecución** | Cómo la solución hace esa tarea más eficiente (qué hace el sistema, qué hace el usuario) |
| **Emociones esperadas** | Cómo se va a sentir el cliente (realista, no utópico) |
| **Beneficios vs situación anterior** | Qué mejora concreta genera esta etapa versus el CJM actual |

**Al cerrar el Paso 2, señalar explícitamente:**
- Qué etapas del CJM actual **se eliminaron** con la nueva solución
- Qué etapas **se simplificaron o fusionaron**
- Si alguna fricción del CJM actual **queda sin resolver** — esto es un riesgo que hay que nombrar
- Si hay **elementos complementarios** que mejorarían la experiencia (tutorial de onboarding, soporte, etc.)

**Regla crítica:** el CJM Futuro debe ser **más simple** que el CJM Actual. Si tiene más etapas, más pasos o más complejidad, la solución está complicando en vez de simplificar. Señalarlo y revisar antes de continuar.

**Lo que NO es CJM Futuro:**
- ❌ Brainstorming libre sin marco
- ❌ Wishlist del emprendedor
- ❌ Listado de funcionalidades disfrazado
- ❌ Promesas emocionales utópicas

Todo lo que no tenga datos del consolidado, marcarlo como supuesto (no eliminarlo, declararlo).

---

### Paso 3 — Comparativa Actual vs Futuro

Contrastar los dos mapas para evidenciar mejoras concretas y cuantificables. Responde la pregunta: **¿es esta mejora lo suficientemente grande para que un cliente cambie su solución actual?**

**Dimensiones a comparar:**

| Dimensión | Qué mide |
|---|---|
| **Tiempo total del proceso** | Cuánto tiempo tarda el cliente de inicio a fin |
| **Costo para el cliente** | Costo directo en dinero y costo de tiempo |
| **Esfuerzo y complejidad** | Cantidad de pasos, herramientas y decisiones involucradas |
| **Satisfacción esperada** | Nivel de satisfacción con el resultado |
| **Fricciones principales** | Las más dolorosas del CJM actual — ¿desaparecen? |
| **Dependencia de terceros** | Cuánto depende el cliente de otras personas o herramientas |

Para cada dimensión: situación actual → situación futura → mejora concreta. **Cuantificar donde sea posible.** "De 4 horas a 15 minutos" es infinitamente más poderoso que "mucho más rápido."

**Al cerrar el Paso 3, evaluar:**
- **El cambio más significativo:** el beneficio más contundente. Una sola cosa, bien articulada.
- **¿Las mejoras son suficientes?** Si ninguna dimensión clave mejora 3x o más, es una señal de riesgo. La solución puede no estar suficientemente diferenciada. Nombrarlo en lugar de ignorarlo.

## Qué sí y qué no

### Sí
- Basar el CJM Actual exclusivamente en evidencia de entrevistas
- Usar citas textuales para soportar las fricciones más importantes
- Identificar explícitamente la etapa más crítica del CJM Actual
- Nombrar las fricciones que el CJM Futuro NO resuelve
- Cuantificar las mejoras de la comparativa donde sea posible
- Señalar como riesgo si ninguna dimensión clave mejora 3x o más

### No
- Construir el CJM Actual desde la imaginación del fundador
- Diseñar un CJM Futuro más complejo que el Actual
- Prometer emociones futuras utópicas
- Avanzar al CJM Futuro sin tener claro el tipo de solución
- Omitir fricciones que la solución no resuelve
- Hacer la comparativa sin cuantificar

---

## Errores típicos

**En el CJM Actual:**
- Incluir soluciones o mejoras durante el mapeo (contamina el diagnóstico)
- Completar huecos "por sentido común" en lugar de respetar solo lo que aparece en entrevistas
- Mapear solo lo funcional y olvidar las emociones
- Tomar el CJM Actual como resumen bonito en lugar de diagnóstico preciso

**En el CJM Futuro:**
- Brainstorming libre sin marco
- Wishlist del emprendedor en lugar de experiencia del usuario
- Listado de funcionalidades disfrazado de CJM Futuro
- No declarar supuestos como supuestos
- Cambiar el recorrido del journey en vez de cambiar cómo se vive cada etapa
- Diseñar mejoras solo funcionales y olvidar las emocionales
